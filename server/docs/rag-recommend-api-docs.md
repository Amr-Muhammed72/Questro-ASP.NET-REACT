# RAG Recommendation Endpoint — Integration Guide

## Overview
This endpoint takes a user's natural-language query and returns a personalized recommendation generated by an LLM. **The frontend should only display the `llmResponse` field to the user.** All other fields are for internal/debugging use.

---

## Endpoint

| | |
|---|---|
| **Method** | `POST` |
| **Path** | `/api/Rag/recommend` |
| **Content-Type** | `application/json` |
| **Auth** | Required — Bearer token |

---

## Authentication

Include a valid bearer token in the `Authorization` header:

```
Authorization: Bearer <token>
```

If the token is missing or invalid, expect a `401 Unauthorized` response.

---

## Request

### Headers
```
Content-Type: application/json
Authorization: Bearer <token>
```

### Body Schema
```json
{
  "query": "string"
}
```

| Field | Type | Required | Description |
|---|---|---|---|
| `query` | string | Yes | The user's free-text input (e.g. "I like adventure games and anime") |

### Example Request
```bash
curl -X 'POST' \
  'https://<host>/api/Rag/recommend' \
  -H 'accept: */*' \
  -H 'Authorization: Bearer <token>' \
  -H 'Content-Type: application/json' \
  -d '{
    "query": "I enjoy action, adventure, and RPG games but not war themes"
  }'
```

---

## Response

### Success — `200 OK`

```json
{
  "query": "string",
  "status": "success",
  "error": null,
  "llmResponse": "string"
}
```

| Field | Type | Description |
|---|---|---|
| `query` | string | Echo of the original query sent by the client |
| `status` | string | `"success"` or `"error"` |
| `error` | string \| null | Error message if `status` is `"error"`, otherwise `null` |
| `llmResponse` | string | **The only field to render to the user.** Plain text containing the recommendation/answer generated by the LLM. May contain `\n` line breaks and markdown-style emphasis (e.g. `**bold**`) — render accordingly or strip markdown if displaying as plain text. |

### Example Response
```json
{
  "query": "I enjoy action, adventure, and RPG games but not war themes",
  "status": "success",
  "error": null,
  "llmResponse": "Hello! As a fellow student who enjoys immersive stories and high-stakes adventure, I've curated a list of games...\n\n1. **The Red Strings Club (Game)**: ...\n2. **G String (Game)**: ...\n\nHappy gaming and watching!"
}
```

### Error Response
If `status` is `"error"`, `llmResponse` may be `null` or empty — display a generic fallback message instead (e.g. "Sorry, something went wrong. Please try again.").

```json
{
  "query": "...",
  "status": "error",
  "error": "Description of what went wrong",
  "llmResponse": null
}
```

---

## Frontend Integration Notes

1. **Only render `llmResponse`** in the UI. Do not show `query`, `status`, or `error` to the end user — these are for logging/debugging.
2. Check `status` before rendering:
   - If `"success"` → render `llmResponse` (convert `\n` to line breaks; handle `**bold**` markdown if your UI supports rich text, otherwise strip it).
   - If `"error"` → show a generic friendly error message, and optionally log `error` to your monitoring tool.
3. Handle HTTP-level failures separately (e.g. `401` → redirect to login, `500` → generic error toast).
4. The response may take a few seconds since it involves an LLM call — show a loading state while waiting.

### Example Integration (JavaScript / Fetch)
```javascript
async function getRecommendation(query, token) {
  const response = await fetch('https://<host>/api/Rag/recommend', {
    method: 'POST',
    headers: {
      'Content-Type': 'application/json',
      'Authorization': `Bearer ${token}`
    },
    body: JSON.stringify({ query })
  });

  if (!response.ok) {
    throw new Error(`Request failed: ${response.status}`);
  }

  const data = await response.json();

  if (data.status === 'success') {
    return data.llmResponse; // <-- only this gets displayed to the user
  } else {
    throw new Error(data.error || 'Something went wrong');
  }
}
```

## Quick Reference

| Action | Field used |
|---|---|
| Display to user | `llmResponse` only |
| Check if call succeeded | `status` |
| Debug/log on failure | `error` |
| Confirm what was sent (optional debug) | `query` |
